既存の React + TypeScript アプリ「ラーメン屋紹介サイト（ラーメン図鑑）」のUIデザインを作ってください。機能・ロジック・APIは変更せず、見た目（CSS とマークアップ構造）のみを作り込んでください。

### サイトの性格
全国のラーメン店を紹介・検索できるサイト。食欲をそそる温かみのある雰囲気で、写真が主役。スマホ・PC両対応（レスポンシブ）。

### デザイン方針
- トーン: 和モダン × 食欲をそそる暖色系。差し色は醤油・味噌を連想させる赤茶系（現状アクセントは #c0392b、自由に変更可）
- 写真を大きく見せるカードUI。余白とタイポグラフィで「専門サイト感」を出す
- 日本語フォント（Noto Sans JP など）前提。見出しはやや太め
- ホバー・トランジションなど軽いマイクロインタラクション歓迎
- アクセシビリティ（コントラスト比、フォーカス可視、alt属性）を確保

### 対象ページとコンポーネント（既存ファイル）
- src/components/Layout.tsx — ヘッダー（ロゴ「🍜 ラーメン図鑑」＋ログイン/ログアウトナビ）、フッター
- src/pages/ShopListPage.tsx — ヒーロー見出し＋検索バー＋店舗カードのグリッド
- src/components/SearchBar.tsx — キーワード入力＋ジャンル/エリアのセレクト
- src/components/ShopCard.tsx — 写真・店名・ジャンル/エリアのタグ・説明・価格帯
- src/pages/ShopDetailPage.tsx — 大きな写真・店名・タグ・説明・住所/営業時間/価格帯
- src/pages/LoginPage.tsx / src/pages/RegisterPage.tsx — 認証フォーム

### 制約（重要）
- 既存の className は維持し、src/index.css のスタイル定義を中心に拡張・刷新してください（必要ならコンポーネントのマークアップ構造の調整は可、ただし props・state・データ取得・ルーティングには触れない）
- 新しいCSSライブラリ（Tailwind等）を導入する場合は提案ベースで。まずは素のCSSで完結する案を優先
- 店舗データのフィールド: name / description / address / area / genre / openingHours / priceRange / imageUrl
- ジャンル例: 醤油・味噌・豚骨・塩・豚骨醤油 / エリア例: 札幌・東京・横浜・博多

まずトップページ（一覧）と店舗詳細のデザイン案を見せてください。
